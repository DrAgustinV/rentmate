RentMate MVP Closeout Plan
1. What "MVP" means here
Single-tenant-per-property B2B SaaS where a manager can: create a property → invite a tenant → sign a contract → collect rent → handle tickets/maintenance → end tenancy. Everything else is a "nice to have."

2. Must-fix before launch (blockers)
Stability & QA
Full end-to-end smoke test on production domain (rentmate.me) for both manager and tenant flows
Verify all email templates render correctly (invitation, verification, password reset, payment reminders)
Check Stripe Connect onboarding works in live mode (currently likely test mode)
Confirm webhook endpoints (Stripe, YouSign) are receiving events on prod
Run Supabase linter, fix any error-level findings

Run security scan, address criticals only
Legal / GDPR (partial today)
Verify Privacy Policy, Terms, Cookie Policy, DPA pages have correct company details (not placeholder text)
Confirm "Export my data" and "Delete my account" both work end-to-end
Cookie consent banner appears for new visitors
Subscription billing


Project knowledge says "Pro/Enterprise not yet available for signup" — decide: enable PRO checkout for MVP, or keep FREE-only and hide pricing CTAs

If FREE-only: hide /pricing PRO button, change to "Contact us"

Critical bugs from recent sessions

4. QA pass checklist (the actual work)
Manager journey (1 hour):
Sign up → verify email → create property → upload photo + document
Run Tenancy Setup Wizard → send invitation
Receive invitation as test tenant → accept → verify ID → sign contract
Generate first month's rent payment → tenant pays via Stripe → manager sees payment
Tenant submits ticket → manager closes it

End tenancy → confirm historic access works
Tenant journey (30 min): mirror of above from tenant side.
Admin journey (15 min): view users, grant a subscription, view analytics, manage carousel.
Edge cases: password reset, account deletion, data export, declined invitation, parallel tenancy transition.
